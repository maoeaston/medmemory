# Key Decisions（≤ 3 条, 影响后续实现）

- **开发阶段无技术债原则**: schema/key/接口变更直接切换, 不写迁移/兼容代码。v3.2 namespace 切换直接重命名 localStorage key（`medmemory:aiApiKey` → `medmemory:ai:ocr:apiKey`）, 不保留旧 key fallback。已存 memory `feedback_no_tech_debt_in_dev.md`
- **v3.2 MVP 限 OpenAI 兼容 provider**: 评审指出 OpenAiProvider 对 Claude API 不兼容（fetch 逻辑 / response 格式不同）, 真要原生支持需 AnthropicProvider + ProviderFactory, 工作量翻倍。MVP 通过中转站切模型即可, 原生 API 推迟 v3.3
- **危急值实时硬规则不落库**: `ai_interpretations.content_json.urgency` 存 LLM 原始值, UI 展示时实时叠加前端硬规则（基于 lab_indicators 实时计算）。这样 lab_indicators 更新后硬规则能正确反映, 不依赖解读时的快照
