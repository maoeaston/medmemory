# Key Decisions（影响后续实现）

- **Repository 接口设计**（`interfaces.ts`）：每表一个 Repo；跨表搜索独立 `SearchRepository`；JSON 字段 TS 层强类型, 实现层序列化；查询返回 null/[], 写入抛错；MVP 不分页
- **sqlite-wasm 走 `sqlite3Worker1Promiser` v2**（worker + OPFS）；不可用时降级 `:memory:`
- **Repository 删 metadata 行不删原件 Blob**：原件删除由 Domain 层显式调 `StorageAdapter.deleteFile(storage_key)`
- **Promiser v2 ESM 调用**：npm 包 `sqlite3Worker1Promiser` 导出本身就是 v2 工厂, 不需 `.v2()` 属性访问
- **Repository 实现层架构**：
  - base.ts 共享 helper（selectOne/selectMany/executeWrite/executeInsertReturningId/readJsonArray/writeJsonArray/fromDbBoolean/toDbBoolean/buildUpdateQuery）
  - 不做 abstract BaseRepository（接口差异大, generic 抽象会过度设计）
  - update 用白名单 buildUpdateQuery + 强制注入 updated_at（与 schema DEFAULT strftime 对齐）
  - create 用 `INSERT ... RETURNING id` 拿自增 PK
  - 无事务: EventProblemRel.attachBatch 循环 exec + TODO 注释
- **SearchRepository 用 LIKE 非 FTS**：MVP 足够; snippet 用 substr+instr
- **UI 层架构**（本次新增）：
  - Repositories 单例通过 useRepositories composable（不引入 feature store）
  - 路由懒加载（code-split 每 view 独立 chunk）
  - Quick Capture 是独立路由（非 modal）, 便于 PWA 直达
  - 三种 capture 共享 KeepAlive 状态（避免误切 tab 丢失未保存输入）
  - 每张 photo 独立一条 inbox_item（schema 设计: inbox_items 是单条记录, 附件等归档时才转 attachment）
